C语言关键字(1)——static

1、全局静态变量

在全局变量前加上static，该全局变量就被定义为全局静态变量。

与全局变量相比，全局静态变量的特点有：

1）未初始化的全局静态变量会被程序自动初始化为0，并且存在静态存储区

2）全局静态变量的作用域为该文件，其它文件不可使用该变量

C语言预处理命令常见问题

一、宏定义#define

宏定义就是用一个字符来表示一个字符串，让程序更简明，同时也便于修改,如#define PI 3.1415926 这样的都很常见，也不会产生什么问题，容易出现问题的就是带参数的宏定义，例如

define S（x） x*x

如果x=10，很明显S(10)=1010；但是如果说x=a+b，此时S(x)=a+ba+b，显然和我们预想的不一样，如果这样定义，则可以达到目的

define S(x) (x)*(x)

当x=a+b时，则S(x)=(a+b)*(a+b)，这样就和预想的一样了。

SignalTap使用（1）

在quartus II中，我们可以使用其软件内部再带的SignalTap II Logic Analyzer，观测器实时信号波形，下面就来说如何使用。

1、将程序编译之后，在菜单栏上点击“Tools”，选择“SignalTap II Logic Analyzer”然后就出现一个新窗口。

2、在弹出的新窗口中，点击Steup（在中级白色区域下面），在Steup区域中，双击，然后出现个“Node Finder”的窗口，在Filter中，可以选择需要什么类的信号，在Look in中，可以选择想要观测的模块的内部信号。然后点击List，会在Nodes Found中列出各种型号管脚或者寄存器，对于我们需要观测的，双击，则会在右边Selected Nodes里面列出，选择完之后，点击OK即可。

3、这时在Steup区域，列出我们需要观测的信号，开始设置触发条件。在“Trigger Conditions”下面选择“Basic”，然后在触发信号后面设置触发条件，不需要触发的，可以将“Trigger Enable”方框中勾去掉即可。

4、设置完之后，在右边的Signal Configuration中，选择Clock，其频率要高于观测信号频率的2倍，在sample depth中选择深度，在Type中选择Continuous即可。
